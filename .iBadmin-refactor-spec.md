# iBadmin 全面重构 spec

## Context（背景与动机）

iBadmin（羽毛球爱好者应用）当前能编译成功（`entry-default-unsigned.hap` 223KB，API 22 / 6.0.1(21)），但存在 **12 项技术债**：

| # | 痛点 | 证据 |
|---|------|------|
| 1 | 硬编码颜色 `#007DFF` 等散落 8 处 | `Index.ets` L47/L51, `ProfilePage.ets` L62 |
| 2 | 硬编码字符串 `'iBadmin'`、`'首页'`、`'社区'` | `Index.ets` L8, `MainTab.ets` L19-L31 |
| 3 | 硬编码数字 `fontSize(20)`、`padding(16)` 50+ 处 | 全页 |
| 4 | **Index.ets 与 MainTab.ets 双入口**（if-else 重复） | `Index.ets` L13-L19 vs `MainTab.ets` L11-L32 |
| 5 | 缺失 MVVM 分层（Page 内嵌业务逻辑） | `ProfilePage.ets` L158-L197 |
| 6 | 缺失 Mock 数据服务 | `HomePage.ets` L3-L22 静态 3 条 |
| 7 | 缺失统一错误处理 | 多处 `console.error` |
| 8 | 0 测试覆盖 | `ohosTest/` 仅有 1 个 stub |
| 9 | 缺失路由系统 | 全部用 if-else 切换 |
| 10 | **AccountService 完全未被消费** | `ProfilePage.ets` L170-L182 自造假登录 |
| 11 | **PrivacyDialog 接口错误**（`@Link onAgree: () => void`） | `PrivacyDialog.ets` L4 |
| 12 | 缺失 DesignTokens 体系 | color.json 定义但未使用 |

按 MediMate（药来）规范体系（73 DesignTokens + 7 铁律 + Superpowers 7 步 + 4 件套开发序），将 iBadmin 升级为 **架构合规、令牌化、可测试、可演进** 的工程基线，保持当前业务功能不变。

## 目标架构

```
View (pages/)          → 纯 UI 展示，绑定 ViewModel
    ↕ 数据绑定 (@State / @StorageProp @Watch)
ViewModel (viewmodel/) → 业务逻辑 + 状态管理，不含 UI 代码
    ↕ 方法调用
Client (cloud/XxxClient.ets) → 领域 API 封装
    ↕ 统一入口
CloudClient (cloud/CloudClient.ets) → 双模式网关 (Mock/Real)
    ↕
MockDataService (cloud/MockDataService.ets)
```

## 编码规范铁律（来自 CONSTITUTION.md）

1. **强制静态类型** — 禁止隐式 `any`，所有对象字面量必须定义接口
2. **禁止 `eval()` / `with` / `obj[key]` 动态属性访问**
3. **严格等号 `===`** 替代 `==`
4. **`throw` 只能抛 Error 实例**：`throw (e instanceof Error) ? e : new Error(String(e))`
5. **字符串用单引号**
6. **`Row/Column` 间距用构造函数 `space` 参数**，禁止 `.gap()`
7. **`ForEach` 必须带第三参数 key generator**：`ForEach(arr, fn, (item) => item.id)`
8. **`$r()` 资源结果禁止 `.toString()`**（Text 组件原生接受 Resource）
9. **组件属性名禁用** `size/width/height/margin/padding/backgroundColor`（与基类冲突）
10. **2 空格缩进，行宽 ≤ 120 字符**
11. **switch case 缩进一层 + 必须大括号**
12. **DesignTokens 强制使用**：所有 `.fontSize(N)` / `.borderRadius(N)` / `.margin(N)` 必须用 Token
